# V0.7 四路线对齐迭代模拟器测试记录

日期：2026-07-20

范围：V0.7 四路线对齐，不进入 V0.8。重点验证“普通应届生 21 天求职行动陪跑”是否继续围绕诊断、行动、记录、复盘、调整闭环。

## 模拟 1：普通应届生，已投递但没有面试

用户画像：2026 届普通本科市场营销学生，投过新媒体运营助理、用户运营助理、市场助理，只有“投了很多但没反馈”的笼统描述。

第一反应：输入页新增“投递记录”后，用户能理解系统不是要直接评价本人，而是先让自己补最近 2-3 条真实投递记录。字段是岗位名称、公司/平台、投递时间、简历版本、反馈状态、岗位要求摘要、对应经历和怀疑的问题，都是普通求职者能看懂的词。

是否知道第一步：知道。信息不足时结果页显示“先补齐记录再复盘”，不再出现基础版报告，也没有把无反馈硬解释成职业方向缺口。

信任点：页面明确说没有完整 JD 时只写真实岗位要求摘要，不确定就留空。这能减少“系统替我编岗位要求”的不信任。

风险点：仍需要用户愿意手动整理记录；但这是 V0.7 的合理边界，不涉及爬虫、自动投递或账号系统。

评分：
- 路线贴合度：5/5
- 第一行动清晰度：5/5
- 真实性边界：5/5
- 继续使用意愿：4/5

结论：通过。当前体验更像“投递复盘陪跑”，不是普通简历报告。

## 模拟 2：无方向、简历没准备好、看到具体岗位的混合检查

用户画像：普通应届生，有课程项目、校园活动和一段轻实习，但方向模糊；另一个场景中贴回真实 JD 想判断这次投递前要改什么。

第一反应：路线入口仍是四个自然问题，而不是专业配置项。无方向路线强调可探索方向和真实岗位样本；简历准备路线强调先整理真实经历；目标岗位路线强调基于真实 JD 和本人材料做投递前修改。

是否知道第一步：
- 无方向：先找真实可搜索岗位样本。
- 简历没准备好：先补一段真实经历的背景、动作、工具、结果。
- 目标岗位：先看当前材料能不能支撑这次投递，再做最小修改。

信任点：页面不显示质量检查、schema、route-specific、字段契约、模型名、token 或成本等内部词；结果页使用“AI 深度建议 / 保守版行动建议 / 演示结果”等用户能理解的身份标识。

风险点：四路线的报告内容仍较长，普通用户可能只看第一屏和今日任务。后续 V0.7 内可以继续压缩结果页首屏，但不需要进入 V0.8。

评分：
- 产品定位一致性：5/5
- 普通用户可读性：4/5
- 行动闭环完整度：5/5
- 不夸大/不承诺结果：5/5

结论：通过。四路线主轴仍围绕行动陪跑，不像招聘平台、CRM 或自动投递工具。

# V0.7 applying_no_feedback 记录对齐修复模拟器测试

日期：2026-07-20
路线：applying_no_feedback
测试目标：验证“已投递但没反馈”不再输出通用职业方向，不再把深度报告误读成基础版兜底。

## 测试背景

用户反馈：
- 已投递但没反馈路线反复输出“用户运营 / 社群运营”“新媒体运营”“教务”，不像根据用户情况定制。
- 自行测试仍看到基础版报告。

本轮修复后重点观察：
- 有 2 条以上结构化投递记录时，深度报告必须绑定岗位名称、简历版本、反馈状态和岗位要求摘要。
- directionOptions 在该路线只作为“投递复盘线索”，不作为职业方向建议。
- rewrites[].jdRequirement 只能来自真实 JD / 用户提供的岗位要求摘要；没有时写“岗位要求缺失，先补 JD 摘要”。
- 页面和反馈表不出现“可探索岗位方向”“建议优先探索方向”等普通方向诊断文案。

## 模拟器测试 1：结构化投递记录深度报告

模拟画像：applying_no_feedback；普通应届生，投了行政助理和采购助理都没反馈，开始怀疑是不是自己不行。

输入样本：
- 行政助理 / Boss 直聘 / 2026-07-10 / 通用版 v1 / 已读未回 / 整理资料、会议支持、Excel 表格。
- 采购助理 / 实习僧 / 2026-07-14 / 行政版 v2 / 无反馈 / 供应商资料整理、表格维护、沟通跟进。
- 用户材料：学院办公室活动报名表、物资清单整理；课程项目供应商资料整理和 Excel 分类汇总。

我看完后的第一反应：这次像是在看我投过的岗位，而不是把我自动归到运营方向。

我现在知道今天先做什么吗：知道。先对照最近 2 条投递记录，检查“岗位要求”和“简历版本”有没有对上，只改 1 条表达。

我觉得它和我有关吗：强相关。报告里出现了行政助理、采购助理、通用版 v1、行政版 v2，也引用了岗位要求摘要。

我会不会继续用：愿意。因为下一轮只验证 1-3 个岗位样本，动作小，不是让我继续盲投。

技术状态和用户价值是否一致：一致。该 run 是 deep report，`isBasic=false`，质量检查通过，同时用户可感知到定制。

深度报告评分：质量 4 / 价值 4 / 可行性 4。理由：路线 fit 明显提升，记录绑定明确；仍可继续补一个可复制表格模板，让用户更快开始填。

哪里让我信任：
- 方向模块变成“先看简历版本和岗位要求是否对上”等复盘线索。
- `jdRequirement` 来自用户填的岗位要求摘要，没有编用户没给的 JD。
- 没有把无反馈归因成“你能力不足”。

哪里让我不信任：
- 如果用户填写记录很散，仍需要更明确的“怎么判断最可疑线索”的例子。
- 简历改写句仍可能沿用模型原句，后续可进一步让改写内容也完全围绕投递记录生成。

像不像普通 AI：有一点像。整体不再像模板方向推荐，但“下一步行动计划”仍可更短、更像陪跑任务。

路线专属判断：通过。它先复盘记录，再给小调整和下一轮小样本验证，没有鼓励海投。

V0.7 红线风险：无明显风险。未新增模型、账号、后台、爬虫、自动投递、支付；未评价用户本人适不适合。

质量与价值结论：通过。

## 模拟器测试 2：结果页与反馈表文案

模拟画像：applying_no_feedback；学生已经很焦虑，如果页面还写“可探索岗位方向”，会觉得系统还是在推荐职业。

我看完后的第一反应：页面标题现在是“投递复盘报告”，正文模块叫“投递复盘线索”，没有把我带回职业方向诊断。

我现在知道今天先做什么吗：知道。页面保留了“最多 3 条真实投递记录”的今日记录入口，并要求写岗位、投递时间、简历版本、反馈状态。

我觉得它和我有关吗：有一点。页面文案符合当前路线；真正强相关还依赖后端报告绑定真实投递记录。

我会不会继续用：勉强愿意。因为反馈表也从“下一步探索方向”换成“下一次投递前先调整什么”，不会在最后一屏混淆路线。

技术状态和用户价值是否一致：一致。前端测试确认 applying 页面不再出现“可探索岗位方向 / 建议优先探索方向”。

深度报告评分：质量 4 / 价值 4 / 可行性 4。理由：页面语义明显对齐投递复盘；但反馈表仍是通用结构，后续可进一步按四路线拆反馈字段。

哪里让我信任：
- 主标题是“投递复盘报告”。
- 摘要强调“不评价你本人”。
- 线索卡字段改成“对应投递记录 / 信息缺口 / 下一轮验证动作”。

哪里让我不信任：
- 如果后端返回的线索名称仍偏职业方向，前端只能改标签，不能完全解决内容质量；本轮已在后端加了记录对齐器降低该风险。

像不像普通 AI：不像原来那么像。少了“建议优先探索方向”这类通用说法。

路线专属判断：通过。页面服务“诊断 - 行动 - 记录 - 复盘 - 调整”闭环。

V0.7 红线风险：无明显风险。

质量与价值结论：通过。

## 迭代建议

1. 高优先级：给 applying_no_feedback 的报告增加一张可复制的 3 行投递复盘表模板，包含“岗位名称 / 岗位要求摘要 / 简历版本 / 投递时间 / 反馈状态 / 可疑线索 / 下一次只改什么”。
2. 中优先级：继续把简历改写内容从“通用改写”收紧到“基于某条投递记录的小修改”，减少模型原始通用句残留。
3. 中优先级：后续四路线反馈表可拆成路线专属字段，但不进入 V0.8，本轮只完成 applying 文案去方向化。

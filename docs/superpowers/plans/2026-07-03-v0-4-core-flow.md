# V0.4 Core Flow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the V0.4 core job-map flow so users always receive a truthful, usable report based on confirmed experiences, with JD-first questioning in JD mode and conservative fallback output when deep AI modules fail.

**Architecture:** Keep the existing React + Express split. First migrate shared TypeScript types and server schemas to V0.4 names and shapes, then update prompt generation and server fallback behavior, then update the single-page UI to enforce the corrected flow and hide internal questioning logic from users.

**Tech Stack:** React 19, TypeScript, Vite, Express 5, Vitest, OpenAI Responses API wrapper currently used as the AI runtime.

---

## Scope

This plan covers the V0.4 P0 core and the minimum P1 support needed for the core flow:

- JD mode requires JD text before experience-card generation.
- JD is structured before dynamic questioning.
- Experience cards must all be processed before continuing.
- Dynamic questioning uses HR/TAR/PART/PREP internally in both modes, but the UI only shows natural questions.
- Reports only use confirmed or edited-confirmed experiences.
- Report modules use V0.4 labels and minimum counts.
- Deep AI failure returns a conservative basic report instead of leaving the user with no result.
- User-facing screens hide token, cost, model details, and internal reasoning labels.

Out of this first implementation batch:

- Real DeepSeek/Qwen/Kimi provider integration.
- Payment, account system, RAG, export enhancements, and historical report comparison.
- Full visual redesign beyond layout fixes required for mobile readability.

- [x] **Step 3: Implement `buildTaskPackage`**

Create `server/taskPackage.ts` with:

```ts
export type TaskPackageModule =
  | 'asset_cards'
  | 'dynamic_questions'
  | 'no_jd_direction'
  | 'jd_match'
  | 'resume_rewrite'
  | 'interview_prep'
  | 'action_plan'
  | 'report';

export function buildTaskPackage(input: Record<string, unknown>) {
  const assets = Array.isArray(input.assets) ? input.assets.filter((item): item is Record<string, unknown> => Boolean(item && typeof item === 'object')) : [];
  const confirmedAssets = assets.filter((asset) => asset.confirmed === true || asset.status === '确认使用' || asset.status === '编辑后确认');
  const unconfirmedAssets = assets.filter((asset) => !confirmedAssets.includes(asset)).map((asset) => String(asset.content || asset.title || '未确认经历'));
  const jdSummary = input.jdSummary && typeof input.jdSummary === 'object' ? input.jdSummary as Record<string, unknown> : {};
  const requirements = Array.isArray(jdSummary.requirements) ? jdSummary.requirements.map(String) : [];

  return {
    meta: {
      version: 'v0.4',
      mode: input.mode === 'jd' ? 'jd' : 'inventory',
      userStage: input.stage === 'junior' ? 'junior' : 'senior',
      module: String(input.module || 'report') as TaskPackageModule,
      language: 'zh-CN'
    },
    userProfile: input.profile || {},
    confirmedAssets,
    excludedAssets: assets.filter((asset) => asset.status === '暂不使用'),
    pendingOrUnverifiedInfo: [],
    userAnswers: [],
    jd: {
      rawText: String(input.jdText || ''),
      structuredRequirements: requirements.map((requirement, index) => ({
        id: `req_${index + 1}`,
        requirement,
        sourceText: requirement
      }))
    },
    currentTask: {
      goal: String(input.goal || ''),
      outputSchemaName: String(input.outputSchemaName || ''),
      qualityRules: [],
      safetyRules: []
    },
    forbiddenInputs: {
      unconfirmedAssets,
      disallowedClaims: ['伪造学历', '伪造公司', '编造数据', '把参与写成负责']
    }
  };
}
```

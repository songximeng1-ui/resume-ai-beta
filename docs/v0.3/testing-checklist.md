# V0.3 测试验收清单

## 功能流程测试

- 开始页可选择阶段和模式。
- 无 JD 模式可完整走到无 JD 报告。
- 有 JD 模式可完整走到有 JD 报告。
- 简历文本可结构化为经历资产卡。
- 经历资产卡可编辑、删除、确认真实。
- 动态追问可保存、跳过、返回。
- JD 匹配可输出证据矩阵。
- 清除本次分析数据可用。

## AI 输出质量测试

- 结构化结果不编造经历。
- 不确定字段标记为待确认。
- 动态追问结合具体经历。
- 有 JD 报告包含证据矩阵。
- 无 JD 报告包含方向建议和补强计划。
- 简历改写可复制、克制、真实。
- 面试追问能覆盖真实性风险。

## 安全与隐私测试

- 不输出伪造学历、公司、证书、时间、数据的建议。
- 不把参与 / 协助写成负责 / 主导。
- 不承诺保证进面或保证 offer。
- 不要求用户输入无关敏感隐私。
- `.env` 不进入 git。
- `.env.example` 不包含真实 Key。
- 前端 bundle 不暴露 API Key。

## UI / 移动端体验测试

- 电脑端全流程无遮挡、无明显布局错位。
- 手机端能完成核心流程。
- 表格或矩阵在移动端可读。
- 长文本不溢出。
- 按钮状态清晰。
- 真实 AI / 演示 / 调用失败状态清晰。

## 真实用户测试

- 用户能否独立完成诊断。
- 用户是否理解每一步要做什么。
- 用户是否觉得报告有帮助。
- 用户是否愿意按补强计划行动。
- 用户是否愿意再次使用或付费。
- 用户在哪一步停留或放弃。

## V0.3 最终验收 Checklist

- [ ] 无 JD 与有 JD 两条流程均可走通。
- [ ] 报告内容符合 AI 输出安全标准。
- [ ] 反馈区可以记录有用性和问题。
- [ ] 构建通过。
- [ ] 自动测试通过。
- [ ] 3-5 个熟人完成测试并给出反馈。
- [ ] 至少 10 个目标用户完成一次诊断。

# V0.3 Web UI 规格

## 产品形态

- 响应式 Web 应用。
- 电脑端完整体验优先，手机端基本可用。
- 首屏直接进入诊断流程，不做营销型 landing page。

## 视觉关键词

- 可信。
- 温和。
- 清楚。
- 克制。
- 有方向感。

## 全局布局规则

- 使用清晰的步骤导航，让用户知道当前在第几步。
- 页面主体宽度控制在适合阅读和填写的范围内。
- 分区使用轻量背景或边框，不使用夸张渐变和成功学视觉。
- 重要操作固定在每页底部或清晰位置。
- 每页只突出一个主任务。

## 页面级 UI 规格

### 开始页

- 使用阶段选择卡和模式选择卡。
- 隐私提示必须可见。
- 主按钮文案明确，如“开始诊断”。

### 输入页

- 简历粘贴框优先。
- 基础信息和经历信息分区。
- 上传简历为占位能力，文案必须说明暂不解析文件。
- 真实性声明在进入资产卡前必须确认。

### 经历资产卡页

- 卡片展示教育、实习、项目、校园、兼职、荣誉、技能作品。
- 缺失经历显示为缺口 / 补强卡。
- 每张卡支持编辑、删除、确认真实。

### 动态追问页

- 显示当前完善第几段经历。
- 追问问题每轮 1-3 个。
- 回答框下方提供保存、跳过、稍后补充。

### JD 匹配页

- JD 粘贴框在上方。
- 证据矩阵用表格或卡片形式展示。
- 投递判断必须醒目但不过度情绪化。

### 报告页

- 使用分块报告结构。
- 每块提供复制友好的内容。
- 极简预览版保持单栏、ATS 友好。

## 状态标签

- 真实 AI 诊断。
- 演示结果。
- 已确认。
- 待用户确认。
- 待核实。
- 估算数据。
- 不建议写入。
- 主投。
- 可冲。
- 过渡。
- 暂不建议主投。

## 按钮层级

- 主按钮：进入下一步、生成报告、分析 JD。
- 次按钮：返回、跳过、稍后补充。
- 危险按钮：删除经历、清除本次分析数据。
- 禁用状态必须说明原因或通过上下文可理解。

## 移动端适配规则

- 单栏布局优先。
- 表格在移动端改为卡片或横向滚动。
- 按钮高度适合触控。
- 文本不被按钮、卡片或固定底栏遮挡。
- 输入框宽度自适应屏幕。

## UI 验收标准

- 电脑端完整流程可走通。
- 手机端能完成输入、确认、追问、查看报告。
- 页面没有夸张营销词。
- 所有核心按钮可点击且状态清晰。
- 长文本不会溢出容器。
- 真实 AI / 演示 / 失败状态清楚显示。

# V0.3 上线与反馈计划

## 第 1 轮：自己测试

- 人数：1 人。
- 目标：发现流程断点、明显 UI 错误、AI 输出安全问题。
- 重点：两条模式都走完，记录每一步是否清楚。

## 第 2 轮：3-5 个熟人测试

- 人数：3-5 人。
- 目标：观察非开发者是否能独立完成诊断。
- 重点：输入成本、追问体验、报告是否可复制。

## 第 3 轮：10-20 个真实目标用户测试

- 人数：10-20 人。
- 目标：验证普通本科 / 双非 / 末流 211 用户是否真的被帮助。
- 重点：报告有用性、方向判断、补强计划可执行性。

## 第 4 轮：20-50 人小范围付费意愿测试

- 人数：20-50 人。
- 目标：验证真实需求强度与付费意愿。
- 重点：用户愿意为什么模块付费、价格敏感点、复购或推荐可能性。

## V0.3 → V0.4 决策规则

- 如果 60% 以上用户能独立完成诊断，继续优化报告质量。
- 如果 50% 以上用户认为报告有具体帮助，推进 V0.4。
- 如果多数用户卡在输入阶段，优先优化输入引导和示例。
- 如果多数用户认为 AI 不准，优先优化 schema、prompt 和追问机制。
- 如果有明显付费意愿，V0.4 再考虑支付或轻量商品化。

## 上线风险

- AI 输出不稳定，可能出现空泛建议。
- 用户输入太少，导致报告质量不足。
- 用户误以为产品承诺 offer。
- 隐私授权说明不清导致信任风险。
- 模型或网络不可用导致真实 AI 失败。

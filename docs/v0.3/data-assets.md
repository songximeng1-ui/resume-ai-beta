# V0.3 数据资产设计

## 用户画像数据

- 当前阶段：准应届生 / 应届生。
- 学历、学校层级、专业、毕业时间。
- 城市意向、目标行业、目标岗位。
- 是否有明确 JD。
- 用户自评：信心、投递经验、简历准备程度。

## 经历资产数据

- 教育背景。
- 实习经历。
- 项目经历。
- 校园经历。
- 兼职经历。
- 荣誉证书。
- 技能与作品集。
- 每条经历的状态：已确认、待用户确认、待核实、估算数据、不建议写入。
- 每条经历的补充追问和用户回答。

## 岗位证据库

- 岗位名称。
- 岗位核心要求。
- 常见能力标签。
- 可迁移经历类型。
- 简历表达示例。
- 面试风险点。
- 相邻岗位方向。

## 诊断结果与反馈数据

- 结构化简历结果。
- JD 证据矩阵。
- 投递判断。
- 简历改写前后对比。
- 面试追问。
- 补强计划。
- 用户反馈：有用性、准确性、行动意愿、最有帮助模块、最不准模块。

## 匿名授权规则

- 默认不公开用户案例。
- 默认不用于训练模型。
- 用户未授权时，不用于公开案例。
- 如需用于案例展示，必须单独授权并脱敏。
- 如未来用于模型训练，必须单独授权。

## 隐私原则

- 不主动收集身份证号、家庭住址、银行卡、家庭成员等无关敏感信息。
- 分享和预览默认脱敏真实姓名、手机号、邮箱。
- 支持清除本次分析数据。
- 复制到第三方平台后，隐私保护由第三方平台规则决定。

## 后续 RAG / 数据库 / 模型训练说明

- 短期不训练模型。
- V0.3 只做结构化数据沉淀。
- V0.4 可考虑建立岗位证据库与案例库，用于 RAG 检索增强。
- 数据库阶段优先存储匿名结构化字段，而不是完整原始简历。
- 模型训练必须在用户单独授权、数据充分脱敏、用途说明清楚后再考虑。

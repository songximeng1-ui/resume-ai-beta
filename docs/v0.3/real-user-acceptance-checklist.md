# V0.3 真实用户验收清单

用于 3-5 个熟人测试和后续 10 个目标用户测试。每次测试建议记录用户阶段、模式、完成情况、报告反馈和明显卡点，不记录身份证号、家庭住址、银行卡号等无关敏感信息。

## 测试对象

- 用户阶段：大三/准应届生、或大四/应届生。
- 用户背景：普通本科、双非、末流 211 等目标用户。
- 测试模式：无 JD 经历盘点 / 有 JD 定制诊断。
- 测试材料：简历文本、经历材料、或目标 JD。

## 流程验收

- [ ] 用户能理解开始页的阶段与诊断方式选择。
- [ ] 用户能完成简历文本输入或粘贴经历材料。
- [ ] 用户能理解真实性确认，不误以为系统会帮助伪造经历。
- [ ] 用户能看懂经历资产卡，并完成确认、修改、暂不使用或补充。
- [ ] 用户能完成动态经历挖掘，且不觉得像考试或审问。
- [ ] 无 JD 用户能进入经历方向诊断，并理解方向建议只是探索起点。
- [ ] 有 JD 用户能进入 JD 证据匹配，并理解它不是录取预测。
- [ ] 用户能生成报告并找到反馈区。

## 报告质量验收

- [ ] 报告至少指出 2 个用户以前没意识到的真实亮点。
- [ ] 简历改写能直接复制，但没有把参与写成负责。
- [ ] 有 JD 报告包含投递判断、JD 证据矩阵和 5 个面试追问。
- [ ] 无 JD 报告包含 2-3 个探索方向和下一步补强建议。
- [ ] 补强计划有周期、动作、产出物和简历用法。
- [ ] 报告没有承诺 offer、保证进面或绝对化人生判断。
- [ ] 内部质量检查显示分数、阻断项和提醒项。

## 用户反馈记录

- 报告帮助评分：1 / 2 / 3 / 4 / 5。
- 最有帮助模块：
  - 发现了我没意识到的亮点
  - 简历改写更清楚
  - 岗位/JD 匹配判断有帮助
  - 面试问题和回答示例有帮助
  - 补强计划具体可执行
  - 语气让我没那么焦虑
- 不准确或不像用户真实情况的地方：
- 看完后是否更清楚下一步：
- 是否愿意修改简历并投递：
- 是否愿意再次使用：
- 是否愿意为完整报告付费：

# V0.6 版本记录

## 2026-07-09：报告复制与 Markdown 导出

改动类型：前端体验、数据结构、测试、文档。

本次完成 V0.6 第三项体验能力：报告复制 / 导出。目标是让小范围真实测试者生成报告后，能直接复制整份报告、复制行动计划，或导出 Markdown 文件用于保存、转发和后续修改。

本次新增：

- 新增报告导出工具：`buildReportMarkdown`、`buildReportText`、`buildActionPlanText`。
- 有 JD 报告、无 JD 报告和通用报告结果页均新增“报告复制和导出”区域。
- 支持“复制整份报告”。
- 支持“复制行动计划”。
- 支持“导出 Markdown”。
- 导出内容包含报告摘要、真实经历亮点、JD 匹配分析或可探索岗位方向、简历改写建议、面试追问、行动计划、信心修复总结、简历正文版、招聘平台字段版、极简预览版和安全提醒。

安全边界：

- 导出内容只基于用户可见报告字段生成。
- 导出内容不包含 provider、模型名、token、成本、base URL、API Key 或内部错误。
- 本次不做 PDF 精美导出、Word 模板导出或历史报告库。

本次测试：

- 新增 `src/reportExport.test.ts`，验证 Markdown / 纯文本导出内容结构和内部信息不泄露。
- 更新无 JD / 有 JD 前端流程测试，验证结果页出现“复制整份报告”“复制行动计划”“导出 Markdown”。

边界：

- 本次未新增环境变量，`.env.example` 暂无必要更新。
- 本次未修改提示词，提示词文档暂无必要更新。
- 本次未修改启动方式，README 暂无必要更新。

边界：

## 2026-07-09：V0.6 启动与验收标准确定

改动类型：文档、迭代边界、验收标准。

本次在 V0.5 provider workflow 收口后，启动 V0.6。V0.6 定位为小范围真实测试版，重点不是新增求职诊断大模块，而是验证真实用户材料下的稳定性、深度报告率、质量通过率和用户反馈。

本次确定：

- 测试人数不超过 10 名。
- 继续使用访问码控制入口，不做账号系统。
- 技术可用率：流程 HTTP 成功率 >= 90%。
- 深度报告率：`isBasic=false` 的报告比例 >= 80%。
- 质量通过率：`quality.passed=true` 的报告比例 >= 90%。
- 需要分别记录 DeepSeek direct 成功率、Qwen fallback 触发次数、Qwen fallback 成功率、失败类型和基础版兜底情况。
- 反馈表必须能表达真实评价，不能只问满意度。
- “哪个模块最没用、不准或帮助较小”必须支持多选，并包含“无”。
- V0.6 支持报告复制 / 导出为 Markdown 或纯文本，暂不做 PDF / Word 精美导出。

V0.6 不做：

- 支付系统。
- 账号体系。
- 历史报告库。
- 复杂后台管理。
- 大规模数据库设计。
- 营销官网。
- 本地部署大模型。

本次新增：

- `docs/v0.6/acceptance-criteria.md`：记录 V0.6 阶段定位、核心验收指标、反馈表设计、体验能力和不做事项。

本次同步：

- README 当前迭代阶段从 V0.5 更新为 V0.6。

边界：

- 本次未改产品功能代码。
- 本次未跑真实 AI smoke。
- 本次未新增环境变量。

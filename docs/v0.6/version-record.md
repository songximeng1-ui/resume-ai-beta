# V0.6 版本记录

## 2026-07-09：V0.6 启动与验收标准确定

改动类型：文档、迭代边界、验收标准。

本次在 V0.5 provider workflow 收口后，启动 V0.6。V0.6 定位为小范围真实测试版，重点不是新增求职诊断大模块，而是验证真实用户材料下的稳定性、深度报告率、质量通过率和用户反馈。

本次确定：

- 测试人数不超过 10 名。
- 继续使用访问码控制入口，不做账号系统。
- 技术可用率：流程 HTTP 成功率 >= 90%。
- 深度报告率：`isBasic=false` 的报告比例 >= 80%。
- 质量通过率：`quality.passed=true` 的报告比例 >= 90%。
- 需要分别记录 DeepSeek direct 成功率、Qwen fallback 触发次数、Qwen fallback 成功率、失败类型和基础版兜底情况。
- 反馈表必须能表达真实评价，不能只问满意度。
- “哪个模块最没用、不准或帮助较小”必须支持多选，并包含“无”。
- V0.6 支持报告复制 / 导出为 Markdown 或纯文本，暂不做 PDF / Word 精美导出。

V0.6 不做：

- 支付系统。
- 账号体系。
- 历史报告库。
- 复杂后台管理。
- 大规模数据库设计。
- 营销官网。
- 本地部署大模型。

本次新增：

- `docs/v0.6/acceptance-criteria.md`：记录 V0.6 阶段定位、核心验收指标、反馈表设计、体验能力和不做事项。
